[00:00 - 01:00] INTRODUCCIÓN
─────────────────────────────
"Buenos días/tardes. Mi nombre es [Nombre] y voy a presentar la defensa técnica de nuestro sistema de microservicios para una plataforma de e-commerce de perfumes.

El proyecto consiste en un backend basado en una arquitectura de microservicios construida con Java 21, Spring Boot 4.0 y Spring Cloud 2025. En total, el sistema está compuesto por 12 microservicios que cubren los dominios de autenticación, usuarios, catálogo, stock, carrito de compras, pedidos, pagos, envíos y notificaciones.

Voy a mostrar la arquitectura general, cómo se despliega con Docker, los endpoints principales, el flujo de compra completo y las pruebas unitarias."

[01:00 - 03:00] ARQUITECTURA GENERAL
──────────────────────────────────────
"La arquitectura sigue el patrón CSR — Client-Service-Registry. Contamos con:

• Eureka Server en el puerto 8761: actuando como servicio de descubrimiento. Todos los microservicios se registran en Eureka al iniciar.

• API Gateway en el puerto 8080: es el punto de entrada único. Todas las peticiones externas pasan por el gateway, que enruta usando el balanceador de carga hacia las instancias registradas en Eureka.

• Los 10 microservicios de negocio son:
  - auth-service (8082): login, registro de credenciales, validación de tokens JWT
  - user-service (8081): CRUD de usuarios y direcciones
  - security-service (8083): roles, permisos y validación de acceso
  - ms-catalogo (8084): marcas, categorías, perfumes y variantes
  - ms-stock (8085): inventario y control de stock
  - ms-carrito (8086): carrito de compras por usuario
  - ms-pedidos (8090): orquestación del flujo de compra
  - ms-pagos (8087): procesamiento de pagos
  - ms-envios (8088): gestión de envíos
  - ms-notificaciones (8089): notificaciones a usuarios

Cada microservicio tiene su propia base de datos MySQL independiente, siguiendo el patrón Database per Service."

[03:00 - 05:00] DEMOSTRACIÓN DEL SISTEMA
───────────────────────────────────────────
"A continuación voy a mostrar el sistema funcionando. Primero, verificamos que todos los contenedores están levantados con 'docker compose ps'. Podemos ver los 13 contenedores — mysql, eureka, gateway y los 10 microservicios — todos en estado 'Up'.

Luego abrimos Eureka en http://localhost:8761 para ver que todos los servicios están registrados y con estado UP.

Ahora vamos a hacer login como administrador. Enviamos un POST a http://localhost:8080/api/auth/login con las credenciales admin@perfume.com / 123456. El sistema responde con un token JWT que contiene los roles ROLE_CLIENTE y ROLE_ADMIN.

Con ese token, podemos consultar el catálogo de productos GET http://localhost:8080/api/catalogo/productos, que nos devuelve los perfumes con sus variantes, precios y stock.

También podemos crear un usuario nuevo, consultar su carrito, agregar items y finalmente confirmar la compra."

[05:00 - 07:00] FLUJO DE COMPRA COMPLETO
───────────────────────────────────────────
"El flujo de compra es orquestado por ms-pedidos. Cuando el cliente hace POST a /api/pedidos, ocurre lo siguiente:

1. ms-pedidos consulta el carrito del usuario en ms-carrito
2. Verifica el stock disponible en ms-stock para cada variante
3. Consulta los precios actualizados en ms-catalogo
4. Crea el pedido en estado CREADO con los detalles
5. Llama a ms-pagos para crear y procesar la transacción
6. Si el pago es exitoso:
   - Reduce el stock en ms-stock
   - Cambia el pedido a PAGADO
   - Crea un envío en ms-envios
   - Envía una notificación en ms-notificaciones
   - Vacía el carrito
7. Si el pago es rechazado:
   - Cambia el pedido a CANCELADO
   - El carrito se mantiene para reintentar

Todo esto ocurre en una sola llamada API, demostrando el patrón de orquestación con Feign Client para la comunicación entre servicios."

[07:00 - 08:30] PRUEBAS UNITARIAS
────────────────────────────────────
"El proyecto incluye pruebas unitarias con JUnit 5, Mockito y MockMvc. En total hay 32 archivos de test con más de 280 métodos de prueba.

Cada microservicio tiene pruebas para sus controladores — probando los endpoints con MockMvc — y para sus servicios — probando la lógica de negocio con Mockito.

Por ejemplo, en auth-service probamos el login exitoso, el login con credenciales inválidas y la validación de tokens. En ms-pedidos probamos la orquestación completa del flujo de compra simulando los clientes Feign.

Además, hemos configurado JaCoCo en el pom.xml padre para generar reportes de cobertura con 'mvn clean verify'. El reporte se genera en target/site/jacoco-aggregate/index.html."

[08:30 - 10:00] DESPLIEGUE CON DOCKER
────────────────────────────────────────
"El sistema está completamente dockerizado. El archivo docker-compose.yml levanta toda la infraestructura:

• MySQL 8.0 en el puerto 3307 con el script de inicialización que crea todas las bases de datos y carga los datos de prueba
• Eureka Server para descubrimiento
• API Gateway como punto de entrada
• Los 10 microservicios

Cada microservicio tiene su propio Dockerfile con construcción en múltiples etapas — primero compila con Maven y luego ejecuta con JRE, lo que optimiza el tamaño de las imágenes.

Para levantar el sistema completo solo se necesita ejecutar 'docker compose up -d'. El tiempo de inicio es de aproximadamente 30-40 segundos."
